Input

Introduction

Inputs are mainly used in Login or Sign-up forms and other general use cases.

Simple Input

With ClutchUI, you can create simple inputs in a jiffy. Just use the input-field in your input class list. You can also define the size of the input field by adding input--md class for medium size.

Input sizes

ClutchUI provides various input sizes. One of which was mentioned above.


There are a total of 3 sizes, which are Small, Medium and Large. You can use the following class to implement them. <input--sm | input--md | input--lg>.

Input shapes

There are 3 input shapes provided by ClutchUI, which are Default, Round and Corner. You can use the following class to implement them. <input--round | input--corner>.


No need to include special class for the Default (sharp corners) input shape.

Input types

There are a total of 5 types of input fields provided by ClutchUI. The input types are as follows.


Primary, Secondary, Success, Warining, Failure. They can be implemented using the following classes.


<input--primary | input--secondary | input--success | input--warning |.
input--fail>

Input Validation

ClutchUI provides 3 types of input validation styles. Validation involves a new <span> tag which includes custom messages.
Validation can be implemented with the input--validation class to the parent container.

Validation successful!
Validation Failed!
You have been warned!

Textarea

ClutchUI also provides styles for textarea.


Use text-area class in your <textarea>.


Additionally you can use text-area--sm, text-area--md and text-area--lg for small, medium and large textareas respectively.
